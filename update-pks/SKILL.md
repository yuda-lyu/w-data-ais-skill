# update-pks:套件更新流程

把使用者的既有手動流程(ncu 查更新 → 手動改 package.json 至最新 → npm i)自動化。**套件分組定義:`wsemi` 與所有 `w-` 開頭之套件為[自有套件],其餘為[非自有套件]**。

**預設更新範圍(重要,使用者裁定 2026-07-23)**:預設**僅更新自有套件**——一律直接更至最新,含 major/破壞性更新,不列待決、不徵詢(自有套件之破壞性更新由使用者自行處理)。**非自有套件預設一律不更動、僅列出**(分「非 major」與「major 跳版」兩節呈現);要更新非自有須明示參數——此設計係防範 vue 2→3、vuex 3→4、eslint 8→10 這類破壞性跳版(殷鑑:rddmanager 為 Vue 2 專案,ncu 會建議 vue ^3.x,盲更即炸),且非自有更新本就非本技能之預設任務。

## 參數

- 無參數:僅自有套件(預設)。
- `others`:自有全部+非自有之非 major 更新。
- `major`:自有全部+非自有全部(含 major 跳版)。
- 其他文字:視為套件名單(空白分隔,可含 glob),只更新命中者(指名者含 major 也更)。

## 步驟

### 1. 前置檢查

- 確認 cwd 有 `package.json`,無則回報並終止。
- **檢查 ncu 可用性**(本技能可能被帶到其他 VM 執行,不可假設已裝):
  ```bash
  ncu --version
  ```
  成功 → 後續以 `ncu` 為指令;失敗 → 改用 `npx -y npm-check-updates`(npx 會自動下載,需網路;首次較慢屬正常)。兩者皆不可用(如無網路)→ 回報無法執行並終止。
- 檢查專案根目錄是否有 `建議*修改.md`(glob):此類檔代表 `node_modules` 內有**尚未收回套件 repo 的本地 patch**,`npm i` 會將其覆蓋。有的話先列出檔名,並於最終回報提醒[哪些本地 patch 已被覆蓋,如仍需要須重套]。**不因此中止**,照常更新。

### 2. 查更新

```bash
<ncu指令> --jsonUpgraded
```

- 輸出為 JSON(`{套件名: 新版本範圍}`)。空物件 `{}` 表示全部已是最新,直接回報並結束。
- 讀 `package.json` 取各套件現行版本,組出 `套件名 現行 → 最新` 對照,並判定各套件是否 major 跳版(比對兩邊 semver 第一碼)。

### 3. 分組呈現

依定義分組列出(各組內按套件名排序),major 跳版者標註:

- **自有套件**:`wsemi` 與 `/^w-/` 命中者。
- **非自有套件(非 major,預設不更新)**:列 `現行 → 最新`,註明須參數 `others` 或指名才更新。
- **非自有套件(major 跳版,預設不更新)**:列 `現行 → 最新`,註明須參數 `major` 或指名才更新。

依參數計算實際更新名單,標明哪些將更新、哪些略過。名單為空則回報後結束(不跑 npm i)。

### 4. 更新 package.json

```bash
<ncu指令> -u --filter "<實際更新名單(空白分隔)>"
```

- 一律用顯式 `--filter` 名單(不整包 `-u`),確保預設僅自有之邏輯生效。

### 5. 安裝

```bash
npm i --no-audit --no-fund
```

- timeout 給足(600000ms);超時改 detached 背景跑再輪詢(依全域 §11.6)。

### 6. 回驗

- `npm i` exit code 為 0 且輸出無 `ERR!`。
- 逐一以 `node -e "console.log(require('./node_modules/<套件>/package.json').version)"` 抽驗本次更新的套件(至少自有套件全部),確認 node_modules 內版本 = package.json 新版本(避免[以為更新了但 node_modules 是舊的]假象)。

### 7. 回報

- 各組更新結果(`套件名 舊 → 新`)、略過清單(含 major 跳版待決清單)。
- npm i 結果與抽驗結果。
- 步驟 1 若有本地 patch 檔:提醒該等 node_modules 改動已被本次安裝覆蓋,如仍需要須重套(列出對應的 `建議*修改.md` 檔名)。
- **不自動 commit**(使用者需要時會自行說)。

## 注意事項

- Windows/git-bash 環境:一律於主任務 cwd 執行,不用 `cd` 前綴。
- 本技能不做相容性評估;非自有清單僅供使用者決策參考,自有更新之回歸驗證由使用者事後把關。
- 曾誤設計為預設連非自有非 major 一起更,經使用者糾正還原(2026-07-23):本技能之本旨即「更新自有套件」,非自有只看不動。
