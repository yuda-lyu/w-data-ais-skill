# L2 主設計原則：濃縮搭配法則

## 1. 字體搭配法則

採用「襯線標題 + 非襯線正文 + 等寬元資料」三分工。

- 標題、金句、數字：襯線，承擔情緒與記憶點。
- 正文、描述、列表、操作說明：非襯線，承擔可讀性。
- kicker、meta、caption、日期、章節、技術標籤：等寬或接近等寬的精準語氣。

字級採固定階層與斷點調整為主，不用 `vw` 直接綁字級，讓文字穩定、可預期、不中途爆版。

## 2. 色彩搭配法則

每個作品只選一套主題色系，再在該色系內建立 light / dark 兩種表面。

- Light：紙面、長文、資料、截圖、可閱讀內容。
- Dark：開場、章節轉場、金句、沉浸式 hero、結尾提問。
- Accent：少量用於標記、焦點、圖表語意，不作大面積裝飾。

不得任意混搭不同主題的 ink 與 paper。若使用者有品牌色，先把品牌色映射到 ink / paper / accent，而不是直接把整頁染成品牌色。

## 3. 版面搭配法則

用「大字 + 網格 + 留白 + 細線」取代卡片堆疊。

- 主訊息靠字級與位置取得權重。
- 次訊息靠欄寬、透明度、字重、細線分隔。
- 圖片與文字以 7:5、8:4、6:6 等明確比例組合。
- 長頁面用 full-width section band，不把整個頁面包進大卡片。
- 卡片只用於重複項目、資料格、工具面板或明確容器，不把卡片再包卡片。

比例與承載對照：

| 比例 | 承載方式 |
|---|---|
| 6:6 | Before / After、雙主角對比、兩組資料 |
| 7:5 | 文字主導 + 輔助圖 |
| 8:4 | 長文主導 + 小圖 / 小數據 |
| 3 欄 | 三支柱、三案例、三個操作 |
| 3 x 2 | 六張截圖、六個 stat、平台矩陣 |
| 全寬 | hero、全景圖片、大引用、章節轉場 |

## 4. 圖像搭配法則

圖片必須承擔敘事功能。

- 優先放真實截圖、產品狀態、場域照片、實際人物、實際圖表。
- 圖片容器使用標準比例：16:9、16:10、4:3、3:2、1:1、3:4。
- 截圖與人物照優先保留頂部與左右資訊；必要裁切時只裁底部。
- 多圖網格統一高度或比例，避免每張圖各自把版面撐亂。
- 圖片缺席時，使用標示明確的 placeholder，不用抽象插圖假裝完成。
- Figure 的 caption 要說明圖片代表什麼，不只重複檔名。

## 5. 節奏搭配法則

讀者的眼睛需要密度變化。

- 每 3-4 個內容段落插入一個節奏點：hero、引用、數據大字報、全寬圖片、章節標記。
- 避免連續三段以上同色、同版型、同密度。
- 長文字後接圖像或數字；密集資料後接留白或大引用。
- 暗色段落不可過多；亮色段落不可一路到底。

## 6. 文案層級搭配法則

每個段落至少分成四個語意角色，且不得互相重複。

- Chrome / Meta：欄目、章節、日期、來源、頁碼感資訊。例：`Act II · Workflow`、`Data · Result`、`Case Study · 2026`。
- Kicker：本段獨有的小鉤子，短、有戲劇性。例：`BUT`、`Phase 01`、`The Question`、`一個人，做了什麼。`。
- Title：主論點或主畫面。
- Lead：一句把標題推進到內容的說明。
- Body：只保留必要細節。

Chrome 與 kicker 不可寫同一句話，也不可互相翻譯。這是避免 AI 味的重要規則。

對應的 HTML 骨架：

```html
<section class="mag-section theme-light">
  <div class="section-meta">Act II · Workflow</div>
  <p class="kicker">The Shift</p>
  <h2>從交接到共建</h2>
  <p class="lead">一句說清這段為何重要。</p>
  <div class="section-body">
    <!-- text / figure / stat / pipeline / comparison -->
  </div>
</section>
```

## 7. 靜態層級搭配法則

靜態網頁的閱讀節奏應由版面、明暗、字級、圖片與留白建立，不依賴程式化特效。

- Hero：用大字、暗色或全寬媒材建立儀式感。
- 正文：用欄寬、標題、lead、細線與段落距離建立層級。
- Quote：用尺寸、留白與出處資訊形成停頓。
- Before / After：用左右欄、透明度、字重或線條建立差異。
- Pipeline：用編號、欄位與分組建立步驟順序。

所有核心資訊必須在純 HTML/CSS 狀態下成立；JavaScript 只能用於非必要的導覽輔助或狀態切換。

## 8. 元件搭配法則

可重用的核心元件如下：

- Big Number：大數字 + 小標籤 + 一句註解。不要每個數字同等權重，最重要的可跨欄。
- Callout：左線或細框引用，不做厚卡片。一段一個觀點，出處用 caption / meta 形式，不用引號裝飾過度。
- Figure：圖片 + caption，caption 像雜誌圖說。
- Pipeline：流程步驟，最多 5 個一列；超過就分組。每步用編號 + 短標題 + 一句描述。
- Before / After：左右對比，舊側弱化，新側提高權重。兩側結構一致；不用紅綠俗套，靠透明度、線條、字重建立差異。
- Image Grid：多張證據圖，以統一比例和圖說建立可信度。
- Hero Question：只放一個問題與一個補句，不塞滿。

## 9. 段落與導覽法則

網頁段落、章節資訊與導覽方式各有固定選擇空間，依內容需求挑選，不混用。

- 段落分兩類：沉浸段落（hero、章節轉場、金句、結尾）與內容段落（正文、資料、案例）。兩類交錯建立節奏。
- 章節資訊（欄目、章節名、來源、頁碼感）放在 section meta、側欄、caption 或頁眉，不塞進主標題或正文。
- 預設使用響應式網格；只有簡報、展示牆、固定畫布等需要每張單獨成立的場景才用 16:9 固定比例。
- 導覽方式有光譜可選：anchor 跳轉、靜態段落順序、橫向 gallery、stepper 進度。依骨架選一種。
- 字級用固定階層與斷點控制；不用 `vw` 直接綁字級，避免文字隨視窗失控（細節見 L5 #1）。

## 10. 素材缺口處理法則

素材不齊時的指導性處理。實際禁止偽造的硬限制見 L5 #8。

| 缺口 | 正確處理 | 錯誤處理 |
|---|---|---|
| 缺圖片 | 使用比例 placeholder，標明所需素材 | 找無關 stock 圖 |
| 缺數據 | 標示待補或請使用者提供 | 編造 KPI |
| 缺 logo | 用文字品牌名或 placeholder | 偽造 logo 牆 |
| 缺 icon | 使用 Lucide 或 `[icon]` | 用 emoji 補洞 |
| 缺文案 | 寫短 placeholder 並列出假設 | 塞 lorem 或空泛行銷語 |
