# L4 骨架原則：依呈現版面選擇結構

## 共通骨架心法

先定「閱讀方式」，再定版型。

- 讀者是一頁頁看：使用 16:9 或 full-screen section。
- 讀者是往下讀：使用長文 / magazine scroll。
- 讀者是掃描大量素材：使用瀑布流 / gallery / masonry。
- 讀者是操作與比較：使用 dashboard / split pane / table。
- 讀者是被說服：使用 landing narrative。

不要把所有內容塞進同一種卡片。骨架要服務閱讀節奏，而不是服務元件庫。

## 1. 16:9 固定畫布

**結構目標**：以固定比例承載一頁一觀點的靜態展示。

**安排原則**：

- 固定一個 16:9 畫布，再讓外層縮放置中。
- 每張畫面只處理一個主觀點。
- 保留雜誌式 chrome：左上欄目、右上頁碼、底部來源或章節。
- Hero 與內容頁交替：封面 / 數據 / 圖文 / 流程 / 引用 / 問題。
- 圖片使用標準比例，避免被視窗底部導覽遮擋。
- 每張畫面要能單獨截圖也成立。

**避免**：

- 每張塞滿段落文字。
- 把網頁 RWD 規則硬塞進固定畫布。
- 連續多張同色同密度。

**響應式簡報引擎 code template**：

純 HTML + JS，不依賴框架，可直接嵌入。關鍵約定：
- 內部陣列 0-indexed，呈現給使用者永遠 1-indexed
- 每個 `<section class="slide">` 加 `data-screen-label="01 Title"` 等標籤以便引用
- 控制按鈕放在 `.stage` 縮放容器外部，確保小螢幕仍可使用

```html
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }
  body {
    background: var(--ink);                          /* 對應 L3 主題色 */
    display: flex;
    align-items: center;
    justify-content: center;
    height: 100vh;
    overflow: hidden;
    font-family: 'IBM Plex Sans', sans-serif;        /* 換成 L2 #1 推薦字體 */
  }
  .stage {
    width: 1920px;
    height: 1080px;
    position: relative;
    transform-origin: center center;
  }
  .slide {
    position: absolute;
    inset: 0;
    display: none;
    padding: 80px;
  }
  .slide.active { display: flex; }
  .controls {
    position: fixed;
    bottom: 20px;
    left: 50%;
    transform: translateX(-50%);
    display: flex;
    gap: 12px;
    z-index: 1000;
  }
  .controls button {
    padding: 8px 16px;
    border: none;
    border-radius: 6px;
    background: rgba(255,255,255,0.15);
    color: white;
    cursor: pointer;
    font-size: 14px;
  }
  .slide-counter {
    position: fixed;
    bottom: 20px;
    right: 20px;
    color: rgba(255,255,255,0.6);
    font-size: 14px;
  }
</style>

<script>
  // 自動縮放適應
  function scaleStage() {
    const stage = document.querySelector('.stage');
    const scaleX = window.innerWidth / 1920;
    const scaleY = window.innerHeight / 1080;
    const scale = Math.min(scaleX, scaleY);
    stage.style.transform = `scale(${scale})`;
  }
  window.addEventListener('resize', scaleStage);
  scaleStage();

  // 投影片導覽
  let current = parseInt(localStorage.getItem('slideIndex') || '0');
  const slides = document.querySelectorAll('.slide');

  function showSlide(n) {
    current = Math.max(0, Math.min(n, slides.length - 1));
    slides.forEach((s, i) => s.classList.toggle('active', i === current));
    localStorage.setItem('slideIndex', current);
    // 呈現給使用者的是 1-indexed，內部儲存為 0-indexed
    document.querySelector('.slide-counter').textContent = `${current + 1} / ${slides.length}`;
  }

  document.addEventListener('keydown', e => {
    if (e.key === 'ArrowRight' || e.key === ' ') showSlide(current + 1);
    if (e.key === 'ArrowLeft') showSlide(current - 1);
  });

  showSlide(current);
</script>
```

## 2. Full-Screen 橫向或縱向段落

**結構目標**：以接近滿版的段落承載故事推進、產品章節或沉浸式報告。

**安排原則**：

- 每個 section 接近一個「螢幕章節」，但不要死鎖 `100vh` 導致小螢幕溢出。
- 使用 `min-height`、安全邊距與內容上限，讓小螢幕能自然捲動。
- 每 3-4 個段落插入 hero 或大引用作呼吸。
- 導覽可用側邊章節、進度點、anchor，避免做成只能逐頁前後翻動的形式。

## 3. 雜誌長文 / Editorial Longform

**結構目標**：以長文節奏承載深度內容、旁欄註解、圖說與章節索引。

**安排原則**：

- 首屏要有主題、作者感、主圖或強烈文字構圖。
- 內文欄寬控制在舒適閱讀範圍，旁欄放來源、註解、圖說、章節索引。
- 使用大首段、pull quote、full-bleed image、數字斷點分割長文。
- 圖說要像雜誌 caption，而非隨便一句 alt 文。
- 內文不要過度裝飾；讓節奏由段落、圖、引用和留白建立。

**長文節奏 recipe**：

1. 首屏：標題、lead、作者 / 日期、主圖。
2. 內文 1：背景。
3. Pull quote：一句核心觀察。
4. 內文 2：分析。
5. Figure：證據圖。
6. Big number：關鍵數據。
7. 內文 3：方法或推論。
8. 結尾：提問或行動。

## 4. 高寬比很大的流式瀑布 / Waterfall Stream

**結構目標**：以流式圖片與節奏錨點承載大量素材、作品、截圖或研究材料。

**安排原則**：

- 用「節奏錨點」避免無止境圖片流：每隔一段插入年份、章節、大數字、引用或小結。
- 瀑布項目可高低不同，但欄寬與間距必須穩定。
- 每個 item 至少有一個短 caption 或 meta，讓圖片有語境。
- 重要項目可跨欄或全寬，形成視覺停頓。
- 圖片載入需保留尺寸，避免 layout shift。
- 大量圖像時優先 lazy loading，並提供篩選或章節跳轉。

**避免**：

- 所有圖片同權重，導致讀者無法判斷重點。
- 用厚卡片和陰影包每張圖，破壞雜誌流。
- caption 太長，讓瀑布流變成碎片文章牆。

**瀑布流節奏 recipe**：

1. 題頭：主題與篩選。
2. 3-6 個 item。
3. 節奏錨點：年份 / 章節 / 數字 / 引用。
4. 6-12 個 item。
5. 跨欄重點 item。
6. 小結或下一組篩選。

## 5. Landing / Product Narrative

**結構目標**：以敘事順序承載說明、證據、方法、結果與行動。

**安排原則**：

- 首屏必須讓產品、品牌或對象成為第一訊號，不只放小 nav。
- Hero 不放在卡片裡；使用真實產品圖、介面圖、照片或沉浸式互動背景。
- 往下用「問題 -> 證據 -> 方法 -> 結果 -> 行動」而非功能卡片堆疊。
- 每個段落只承擔一個說服任務。
- CTA 要像編輯指令，清楚、少量、有位置權重。

**8 段網頁敘事 recipe**：

1. Hero dark：一句主張。
2. Light：大數據或背景。
3. Light / dark split：故事或問題。
4. Light：流程或方法。
5. Hero light：章節呼吸。
6. Dark：金句或對比。
7. Light：證據牆或案例。
8. Hero dark / light：問題、CTA 或收束。

## 6. Dashboard / Data Story

**結構目標**：以安靜、密集、可掃描的方式承載資料、表格、圖表與操作狀態。

**安排原則**：

- 回到安靜、密集、可掃描的工具感，不做過度 hero。
- 大數字、趨勢圖、明細表分層，不讓所有指標同時喊叫。
- 顏色承載語意：增減、風險、分類、狀態。
- 用 data-ink ratio 思維刪除多餘格線、陰影、裝飾。
- 保留 filter、空狀態、loading、error、hover、focus 等操作狀態。

**庫選擇**：

| 條件 | 推薦 | 理由 |
|---|---|---|
| 儀表板、複雜互動、大資料集 | ECharts（預設） | 2000+ 圖表、內建 zoom/pan/tooltip、百萬級資料點、官方 dark theme |
| 小規模、快速拋棄式原型 | Chart.js | 輕量、API 最簡單，可承載單一 KPI 卡 |
| 完全客製、藝術向資料藝術 | D3.js | 最底層，學習曲線陡，表現力最強 |

不確定時選 ECharts。

### ECharts code template

ECharts/Chart.js 不直接讀 CSS variables，需用 `getComputedStyle` 讀 L3 token 後傳入：

```html
<div id="chart" style="width: 100%; height: 400px;"></div>
<script src="https://cdn.jsdelivr.net/npm/echarts@5.5.1/dist/echarts.min.js"></script>
<script>
  const root = getComputedStyle(document.documentElement);
  const ink       = root.getPropertyValue('--ink').trim();
  const accent    = root.getPropertyValue('--accent').trim();
  const paperTint = root.getPropertyValue('--paper-tint').trim();

  // 第二參數為主題：'default' | 'dark' | 自訂 theme 名稱
  const chart = echarts.init(document.getElementById('chart'), 'default');

  const option = {
    tooltip: { trigger: 'axis' },
    legend: { show: false },
    grid: { left: 40, right: 20, top: 20, bottom: 30 },
    xAxis: {
      type: 'category',
      boundaryGap: false,                                   // line chart 慣例：緊貼軸端
      data: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'],
      axisLine:  { lineStyle: { color: paperTint } },
      axisLabel: { color: ink }
    },
    yAxis: {
      type: 'value',
      splitLine: { lineStyle: { color: paperTint } },
      axisLabel: { color: ink }
    },
    series: [{
      name: 'Revenue',
      type: 'line',
      smooth: true,
      data: [12, 19, 3, 5, 2, 3],
      lineStyle: { color: accent, width: 2 },
      itemStyle: { color: accent }
    }]
  };
  chart.setOption(option);

  // 響應式：容器尺寸變動時重繪
  new ResizeObserver(() => chart.resize())
    .observe(document.getElementById('chart'));
</script>
```

**Dark Mode 切換**（配合 `prefers-color-scheme` 或 Tweaks 切換，ECharts 切主題需重建實例）：

```js
function applyTheme(isDark) {
  chart.dispose();
  const newChart = echarts.init(
    document.getElementById('chart'),
    isDark ? 'dark' : 'default'
  );
  newChart.setOption(option);
  return newChart;
}

window.matchMedia('(prefers-color-scheme: dark)')
  .addEventListener('change', e => applyTheme(e.matches));
```

**常用配置要點**：
- `tooltip.trigger: 'axis'`：折線／長條圖用，hover 軸上任意位置顯示所有 series
- `tooltip.trigger: 'item'`：散點／圓餅圖用，hover 單一資料點
- `dataZoom: [{ type: 'inside' }, { type: 'slider' }]`：時序大資料集必備
- `toolbox.feature.saveAsImage`：匯出 PNG
- 大資料集：`large: true, largeThreshold: 2000` 啟用優化渲染

### Chart.js code template（輕量替代）

```html
<canvas id="myChart" width="800" height="400"></canvas>
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.7/dist/chart.umd.min.js"></script>
<script>
  const root = getComputedStyle(document.documentElement);
  const accent    = root.getPropertyValue('--accent').trim();
  const paperTint = root.getPropertyValue('--paper-tint').trim();

  const ctx = document.getElementById('myChart').getContext('2d');
  const chart = new Chart(ctx, {
    type: 'line',                           // bar、pie、doughnut、radar 等
    data: {
      labels: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'],
      datasets: [{
        label: 'Revenue',
        data: [12, 19, 3, 5, 2, 3],
        borderColor: accent,
        backgroundColor: 'rgba(0,0,0,0.05)',
        tension: 0.4,
        fill: true
      }]
    },
    options: {
      responsive: true,
      plugins: { legend: { display: false } },
      scales: {
        y: { beginAtZero: true, grid: { color: paperTint } },
        x: { grid: { display: false } }
      }
    }
  });

  new ResizeObserver(() => chart.resize())
    .observe(document.querySelector('#myChart').parentElement);
</script>
```

## 7. Split Narrative / 圖文混排

**結構目標**：以明確比例承載圖文敘事、對比、流程拆解或雙主角內容。

**安排原則**：

- 7:5：文字主導、圖片輔助。
- 8:4：長文主導、小圖或數據輔助。
- 6:6：對比、雙主角、Before / After。
- 左側文字若需要底部 callout，用 flex column 分配；不要把右側圖片硬貼底。
- 圖片貼頂、文字自行形成上下節奏，避免底部被導覽或視窗裁掉。

## 8. Mobile / Narrow View

**結構目標**：讓所有窄螢幕版本保留完整語意順序與可讀性。

**安排原則**：

- 多欄收斂為單欄，但保留 kicker -> title -> lead -> media -> body 的語意順序。
- Hero 字級縮小，不使用造成單字爆行的超大標題。
- 圖片放在相關文字附近，不把所有圖片堆到段落後方。
- 觸控目標至少 44px。
- 固定式 chrome、導覽與浮動控制不可遮住內容。
