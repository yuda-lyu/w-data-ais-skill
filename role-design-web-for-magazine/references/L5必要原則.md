# L5 必要原則：產製設計稿件的硬限制

## 1. 字級與可讀性

- 網頁正文最小 16px；長文建議 17-18px。
- Caption / meta 最小 12px；若資訊重要，不可低於 13px。
- 行高：正文 1.55-1.75；lead 1.35-1.55；大標 1.05-1.2。
- 一般網頁不要用原始 `vw` 直接控制字級；用固定階層、斷點與容器寬度調整。
- 文字不可出現一字一行、按鈕文字溢出、卡片文字被裁切。
- `letter-spacing` 預設為 0；不要用負字距。若需要雜誌式元資料感，優先用字重、大小寫、留白與排版位置處理。

## 2. 對比與明暗

- 正文對比至少符合 WCAG AA：一般文字 4.5:1，大字 3:1。
- 暗色 hero 可以有靜態背景材質或圖像，但文字區必須有足夠遮罩或留白。
- 長時間閱讀內容放在亮色紙面，暗色只用於節奏點。
- 不要讓 paper 色過灰、過黃或過暗，否則會像套了髒濾鏡。
- Light / dark 切換時，border、icon、caption 必須跟著 `currentColor` 或 token 適配。

## 3. 色彩限制

- 一個作品只使用一套主題色，不中途換 palette。
- 不任意接受使用者提供的單一 hex 作為整體主題；需映射為完整 token。
- 不混搭不同主題的 `ink`、`paper`、`paper-tint`。
- 避免整頁單一色相變體；即使是紙色主題，也要靠深墨、圖片、留白與細線建立層次。
- Accent 只用於焦點、資料語意或少量標記，不作大面積背景。

## 4. 圖片與媒材

- 圖片容器只能使用標準比例：16:9、16:10、4:3、3:2、1:1、3:4。各情境建議：
  - 截圖：優先 16:10 或 4:3。
  - Hero 主視覺：16:9 或 full-bleed。
  - 圖文主圖：16:10 或 4:3。
  - 案例：3:2 或 1:1。
  - 直式輔助：3:4。
  - 小卡 / 頭像 / logo placeholder：1:1。
- 截圖、人物、產品圖優先保留頂部與左右；必要時只裁底部。
- 多圖網格使用統一高度或統一比例，避免素材各自拉扯版面。
- 圖片要有 width / height、aspect-ratio 或預留尺寸，避免 layout shift。
- 缺素材時使用 placeholder，並標明需要的素材種類與比例；不可用假素材冒充。

## 5. 版面與間距

- 所有固定格式元素要有穩定尺寸：棋盤、工具列、資料格、圖片框、按鈕、卡片。
- 不使用巢狀卡片；頁面區塊用 full-width band 或無框布局。
- 卡片圓角最多 8px；圖片微圓角約 4px。
- 不使用厚陰影；需要層級時用線條、色差、間距與排版。
- 不讓 UI 元素互相重疊，尤其是浮動導覽、底部控制、caption、圖片。
- 避免硬設 `height:100vh` 造成瀏覽器工具列吃掉內容；使用 `min-height`、safe area 與內容上限。

## 6. 元件與 icon

- 預設不用 emoji 當 icon。
- 若已有 icon library，沿用既有系統；沒有時優先使用 Lucide 或明確 placeholder。
- 按鈕使用熟悉符號或 icon 表達常見操作，並提供 hover/focus/active/disabled 狀態。
- Kicker、chrome、caption、foot 不得互相重複同一句話。
- Pipeline 單列最多 5 個步驟；更多步驟要分組或換成時間線。
- Big Number 數字不要過長，必要時用 K / M / % 縮寫。

## 7. 靜態互動與腳本

- 不使用特效函式庫作為設計核心。
- 不以程式化效果建立資訊層級；層級必須由字級、網格、色彩、留白與圖片完成。
- JavaScript 只用於必要的靜態站輔助，例如導覽開合、篩選、頁籤或狀態切換。
- JS 失敗時，所有核心內容必須仍可閱讀與導覽。
- 不使用滾動觸發特效、逐字特效或依賴程式化效果才理解的內容。

## 8. 內容倫理

- 不偽造數據、logo、使用者評價、合作品牌。
- 不用無意義統計數字填空。
- 沒有資料就標示缺口或請求資料。
- 每個區塊都要回答「為何存在」；答不出來就刪掉。

## 9. 響應式與驗收

交付前至少檢查：

- Desktop、tablet、mobile 三種寬度。
- 文字是否溢出、重疊、被固定導覽遮住。
- 圖片是否裁到關鍵資訊。
- Light / dark 對比是否足夠。
- hover、focus、active、disabled、loading、empty、error 是否符合需求。
- Console 無錯誤。
- 不執行 JavaScript 或低效能環境下內容仍完整。
